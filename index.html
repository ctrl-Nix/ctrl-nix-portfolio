01
WHAT IS A RUBBER DUCKY?
THE BASICS // BEGINNER
INTRO
Imagine you find a USB drive on the ground. You plug it in. But instead of files, it's actually a tiny computer pretending to be a keyboard. Within seconds, it types commands into your computer — faster than any human — and installs software, steals files, or creates backdoors.

The victim sees nothing. The OS trusts it completely. Because keyboards are trusted devices.
device_type: USB Keyboard (trusted)
threat_scan: CLEAN — no malware detected
reality: IT'S A RUBBER DUCKY. YOU'VE BEEN HAD.
02
THE PAYLOAD SCRIPT
DUCKY SCRIPT LANGUAGE // HOW IT TALKS
CODE
Rubber Ducky runs a simple script language called DuckyScript. Each line is an instruction — like DELAY (wait), STRING (type text), ENTER (press enter). The device executes this script the moment it's plugged in.
DELAY 500 // wait 500ms for OS to recognize device
GUI r // press Windows + R (Run dialog)
DELAY 300
STRING cmd.exe // type "cmd.exe"
ENTER
DELAY 400
STRING whoami && ipconfig /all // get identity + network info
ENTER

HOW TO DEFEND YOURSELF
PROTECTION STRATEGIES // CRITICAL
DEFENSE
Now that you know the attack — here's how ctrl-Nix would defend against it. Good security isn't about paranoia. It's about knowing your threat model and taking specific, targeted steps.
✓ NEVER plug in unknown USB devices // #1 rule
✓ Enable USB device whitelisting // only known devices allowed
✓ Use endpoint protection that blocks HID attacks
✓ Disable USB ports in BIOS for sensitive machines
